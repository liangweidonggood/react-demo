# React Router v7 导入方式更新计划

## 问题分析

React Router v7已经合并了`react-router`和`react-router-dom`包，推荐直接从`react-router`导入所有内容，而不是从`react-router-dom`导入。

## 解决方案

更新项目中所有从`react-router-dom`的导入，改为从`react-router`导入。

## 实施步骤

1. **更新package.json**
   - 移除`react-router-dom`依赖
   - 添加`react-router`依赖

2. **更新导入语句**
   - 修改`src/App.tsx`：将`import { RouterProvider } from 'react-router-dom'`改为`import { RouterProvider } from 'react-router'`
   - 修改`src/routes/routeConfig.tsx`：将`import { createBrowserRouter } from 'react-router-dom'`和`import type { RouteObject } from 'react-router-dom'`改为从`react-router`导入
   - 修改`src/pages/Layout.tsx`：将`import { Outlet, useNavigate } from 'react-router-dom'`改为从`react-router`导入
   - 修改`src/pages/Login.tsx`：将`import { useNavigate, useLocation } from 'react-router-dom'`改为从`react-router`导入
   - 修改`src/components/ProtectedRoute.tsx`：将`import { Navigate, Outlet, useLocation } from 'react-router-dom'`改为从`react-router`导入

3. **测试项目**
   - 运行`pnpm build`确保项目能正常构建
   - 运行`pnpm dev`确保开发服务器正常运行

## 预期效果

- 项目使用React Router v7的推荐导入方式
- 代码更符合v7的设计理念
- 减少不必要的依赖

## 注意事项

- 虽然v7仍然支持从`react-router-dom`导入，但推荐使用从`react-router`直接导入的方式
- 本次更新不会改变项目的功能，只会更新导入方式
